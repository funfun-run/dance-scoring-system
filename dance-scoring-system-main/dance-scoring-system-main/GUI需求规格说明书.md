# 舞蹈评分系统 GUI 界面需求规格说明书

## 一、项目背景

当前舞蹈评分系统（`score_dance.py` v6.2 + `split_8beats.py` v2.3）为命令行工具，用户需通过终端输入命令完成视频分段、评分和结果查看。为提升易用性，需构建图形化界面（GUI），将核心流程（导入视频 → 预览 → 分段 → 评分 → 查看结果）整合为可视化操作。

**基础代码**：`dance-scoring-system-main/dance-scoring-system-main/`

---

## 二、技术选型

| 方案 | 优点 | 缺点 |
|------|------|------|
| **PyQt6** | 功能完善、原生视频播放、丰富控件 | 需额外安装、包体积大 |
| **Tkinter** | Python 内置、零额外依赖、轻量 | 视频播放需借助 OpenCV |
| **Flask + Web** | 跨平台、前端灵活 | 启动服务、不适合单机场景 |

**推荐方案**：**Tkinter + OpenCV**。理由：项目已有 OpenCV 依赖，无需新增 GUI 框架依赖，适合本地单机使用场景。

## 三、功能需求

### 3.1 主界面布局

```
┌──────────────────────────────────────────────┐
│  舞蹈评分系统 v6.2                            │
├──────────────┬──────────────┬────────────────┤

│  ┌─────────┐ │ ┌─────────┐ │                │
│  │         │ │ │         │ │ [导入参考视频]   │
│  │ 预览区  │ │ │ 预览区  │ │ [导入用户视频]   │
│  │         │ │ │         │ │                │
│  └─────────┘ │ └─────────┘ │ BPM: [120]     │
│              │              │ 阈值: [50]     │
│  参考视频名   │  用户视频名   │                │

```

### 3.2 视频导入 (P0)

**需求描述**：通过 GUI 界面导入参考视频和用户视频。

**交互流程**：
1. 点击「导入参考视频」按钮 → 弹出文件选择对话框 → 过滤常见视频格式（`.mp4`, `.avi`, `.mov`, `.mkv`）
2. 点击「导入用户视频」按钮 → 同上
3. 导入后：
   - 显示视频文件名（截断过长的路径）
   - 显示视频时长（秒）、分辨率（宽×高）、帧率
   - 在左侧预览区显示视频第一帧缩略图
   - 视频路径存储供后续处理使用

**异常处理**：
- 文件不存在 → 弹窗提示
- 文件无法被 OpenCV 打开 → 弹窗提示"不支持的文件格式"
- 取消选择 → 保留原路径不变

**复用代码**：`split_8beats.py:get_video_info()` — 已有视频信息提取函数

### 3.3 视频预览播放 (P1)

**需求描述**：在 GUI 内预览已导入的视频。

**交互流程**：
1. 点击「预览播放」按钮 → 对应侧预览区开始播放视频
2. 播放中：按钮变为「停止播放」→ 再次点击停止
3. 使用 OpenCV 逐帧读取 + Tkinter Canvas/Panel 渲染
4. 按原始帧率播放（30fps）或降低至 15fps（节省 CPU）
5. 两侧可独立播放互不影响

**技术方案**：
- 使用 Tkinter `after()` 定时器驱动帧刷新
- 帧缩放到预览区尺寸（保持宽高比）
- 循环播放（播完自动从头开始）
- 切换视频前自动停止当前播放

### 3.4 视频分割与预览 (P0)

**需求描述**：执行 `split_8beats.py` 逻辑，并在 GUI 中查看分割后的慢动作片段。

**交互流程**：
1. 用户配置 BPM（默认 120）
2. 点击「分割参考视频」→ 后台调用分段逻辑
3. 进度条显示处理进度
4. 完成后弹出「分割结果」窗口：
   - 列表显示所有段号、时间段、文件名
   - 点击任一段 → 右侧预览区播放该段慢动作视频
   - 支持逐段播放 / 停止
5. 分割结果保存在 `output/segments/` 目录

**复用代码**：直接调用 `split_8beats.py` 的函数：
- `get_video_info()` — 视频信息
- `detect_beats_from_audio()` — 音频节拍检测
- `detect_beats_from_motion()` — 运动检测
- `calculate_segments_fixed()` — 固定 BPM 分段
- `extract_slow_segment()` — 慢动作提取
- `merge_videos()` — 合并

### 3.5 评分与结果展示 (P0)

**需求描述**：执行评分流程，在 GUI 中展示完整评分结果。

**交互流程**：
1. 用户确认已导入参考视频 + 用户视频
2. 配置 BPM（需与分割时一致，默认 120）和阈值（默认 50）
3. 点击「开始评分」→ 后台执行：
   - 步骤 1/3：提取参考视频姿态（进度条更新）
   - 步骤 2/3：提取用户视频姿态（进度条更新）
   - 步骤 3/3：DTW 对齐 + 逐帧评分 + 分段评分（进度条更新）
4. 完成后弹出「评分结果」窗口：

```
┌──────────────────────────────────────────────┐
│  评分结果                                     │
├──────────────────────────────────────────────┤
│  总评: 62.5/100  ⚠️需改进                     │
│  参考帧数: 571  用户帧数: 571  DTW对齐: 604对  │
│  BPM: 120  合格线: 60分                       │
├──────┬──────────┬────────┬───────────────────┤
│ 段号  │ 时间      │ 得分    │ 判定               │
├──────┼──────────┼────────┼───────────────────┤
│  1   │ 0.0-4.0s │  58.3  │ ❌不合格            │
│  2   │ 4.0-8.0s │  69.8  │ ✅合格              │
│  ... │ ...      │  ...   │ ...               │
├──────┴──────────┴────────┴───────────────────┤
│  ❌ 2段不合格，已输出练习视频                    │
│  [打开练习视频目录]  [打开分段目录]              │
└──────────────────────────────────────────────┘
```

**复用代码**：直接调用 `score_dance.py` 的核心类：
- `PoseExtractor` — 姿态提取（需进度回调）
- `Scorer` — 评分引擎
- `Scorer._seg_by_beats()` — 分段评分
- `Scorer.extract_clips_from_segments()` — 练习片段提取
- `Scorer._grade_overall()` — 总评计算

### 3.6 练习视频查看 (P1)

**需求描述**：查看和播放需练习的低分/不合格片段。

**交互流程**：
1. 评分完成后，不合格/低分段自动显示在评分结果中
2. 点击「查看练习视频」→ 弹出练习视频列表窗口
3. 列表显示：段号、得分、文件名
4. 双击或点击播放 → 播放对应慢动作练习视频
5. 支持用系统默认播放器打开（`os.startfile` on Windows）
6. 练习视频位于 `output/low_score_clips/`

## 四、非功能需求

### 4.1 性能要求
- 视频预览帧率 ≥ 15fps（720p 视频）
- 姿态提取进度刷新间隔 ≤ 500ms
- GUI 在处理期间保持响应（后台线程执行耗时操作）

### 4.2 兼容性
- Windows 10/11 优先
- Python 3.10+ 
- 分辨率自适应（最低 1280×720）

### 4.3 可用性
- 所有耗时操作有进度提示
- 操作按钮在任务执行期间禁用，防止重复操作
- 状态栏实时显示当前操作状态
- 错误信息弹窗友好提示

## 五、代码改动范围

### 5.1 新建文件

| 文件 | 说明 |
|------|------|
| `gui_main.py` | GUI 主程序入口，主窗口布局与交互逻辑 |
| `gui_components.py` | 可复用 GUI 组件（视频预览面板、进度条对话框、结果表格） |
| `gui_worker.py` | 后台线程封装（姿态提取、DTW 评分、视频分割），支持进度回调 |

### 5.2 需改动的现有文件

| 文件 | 改动点 | 说明 |
|------|--------|------|
| `score_dance.py` | `PoseExtractor.extract()` 增加 `progress_callback` 参数 | 支持 GUI 进度更新 |
| `score_dance.py` | `Scorer.score()` 增加 `progress_callback` 参数 | DTW/评分阶段进度回调 |
| `requirements.txt` | 无需新增依赖 | Tkinter 为 Python 内置 |

**改动原则**：对现有代码最小侵入。progress_callback 为可选参数（默认 None），CLI 模式不受影响。

### 5.3 不改动的文件

- `split_8beats.py` — 通过模块导入直接调用，不改动
- `check_env.py` — 保持不变

## 六、实现顺序

| 阶段 | 内容 | 预估工作量 |
|------|------|-----------|
| Phase 1 | `gui_main.py` — 主窗口框架 + 视频导入 + 信息显示 | 基础 |
| Phase 2 | `gui_components.py` — 视频预览播放组件 | 中等 |
| Phase 3 | `gui_worker.py` — 后台线程 + 进度回调 | 中等 |
| Phase 4 | 分割流程 GUI 集成 + 分段结果查看 | 中等 |
| Phase 5 | 评分流程 GUI 集成 + 结果展示 | 中等 |
| Phase 6 | 练习视频查看 + 异常处理完善 | 较小 |
| Phase 7 | 整体测试 + 边界情况修复 | 较小 |
