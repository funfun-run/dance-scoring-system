P0
DTW 对齐缺少窗口约束
影响：对齐质量、分段准确性
现象

当前 _dtw 方法是标准的无约束 DTW，允许参考视频第 1 帧对齐到用户视频第 500 帧——物理上不可能，但算法允许。实际运行输出中出现了多个 0.1 秒长的段落（如 10.1s–10.1s），就是 DTW 路径在某一段"卡住"的典型症状。

涉及代码
# score_dance.py:150-170  _dtw() 方法
# 无 warping window 约束，允许任意 |i-j| 的路径
建议方案
  • 加上 Sakoe-Chiba band 约束，限制 |i - j| ≤ window_size
  • 窗口大小可取 max(nr, nu) * 0.1（允许 10% 的时间伸缩）
  • 也可用 scipy.spatial.distance.cdist 向量化距离矩阵计算，避免纯 Python 双循环的性能瓶颈
  • 加入 Itakura parallelogram 作为进阶选项，对舞蹈这种有节奏约束的场景更合适
P0
仅使用 12 个关节角度，遗漏大量舞蹈关键关节
影响：评分维度完整性
现象

当前 ANGLE_JOINTS 只定义了 12 个角：左右肘、左右膝、左右髋-踝线、左右肩-髋角度。MediaPipe 提供了 33 个关键点，利用率不到 40%。大量对舞蹈表现力至关重要的关节完全缺失：

缺失的关节
缺失维度 MediaPipe 关键点 舞蹈意义
脊柱前倾/侧弯 (11,23,24), (12,24,23) Body wave、胸腰 isolation
肩关节外展/内收 (13,11,23), (14,12,24) 手臂框架、对称性评判
髋关节矢状面 (11,23,25), (12,24,26) 踢腿高度、控腿稳定性
腕关节 (13,15,19) 等 手势细节、手指方向
颈部/头部朝向 (7,8,0), (7,8,11) 头部方向、表现力
踝关节 (25,27,31), (26,28,32) 绷脚尖/勾脚尖、足部姿态
建议方案
  • 扩充 ANGLE_JOINTS 到 20–24 个角，覆盖上述缺失维度
  • 对快速关节（腕、踝）和慢速关节（躯干、髋）分别设置不同的变化容忍阈值
P1
使用 world landmarks 的深度精度差
影响：姿态估计基础质量
现象

score_dance.py:91 使用 res.pose_world_landmarks，这是 MediaPipe 从单目 RGB 图像估算的 3D 世界坐标。单目深度估计本身误差较大（z 轴尤其不可靠），而舞蹈动作中的快速旋转、肢体交叉会进一步放大误差。当舞者侧身或背对镜头时，z 轴数据几乎不可用。

涉及代码
# score_dance.py:91-98
res = self.det.detect_for_video(img, ts)
if res.pose_world_landmarks and len(res.pose_world_landmarks) > 0:
    for i, lm in enumerate(res.pose_world_landmarks[0][:33]):
        kp[i] = [lm.x, lm.y, lm.z]  # z 轴噪声大
建议方案
  • 方案 A（推荐）：改用 pose_landmarks（图像 2D 归一化坐标）计算角度，牺牲深度信息但大幅提升 x/y 平面精度
  • 方案 B：同时使用 world + image landmarks，对 z 轴分量做加权衰减（如 0.3× z + 1.0× x + 1.0× y）
  • 可选的进阶方案：利用 MediaPipe 的 pose_world_landmarks 分段对齐器（segmented alignment）减少漂移
P1
评分函数过于线性且无关节权重
影响：评分与主观感受的一致性
现象

score_dance.py:141 使用 s = 100 - diff * 1.0，对 12 个角度的均值做线性罚分。这意味着：

  • 肘关节差 5° = 膝关节差 5°，但视觉上膝关节偏差远比肘关节显眼
  • 舞蹈"关键帧"的定格造型和过渡帧获得相同权重
  • 均值回归效应导致用户总是拿到中等分数，区分度不足
建议方案
  • 为不同关节设置差异化权重：腿部 > 躯干 > 手臂 > 腕/踝
  • 引入非线性罚分：小偏差宽容（如 < 5° 不扣分），大偏差加速扣分（sigmoid 或分段函数）
  • 识别舞蹈中的关键姿势锚点（pose anchors），在锚点帧加倍权重
  • 增加时间一致性维度：连续多帧偏差大（姿势不稳定）额外扣分
P2
动作变化检测硬切为 8 段，与编舞结构脱节
影响：分段合理性
现象

score_dance.py:172-187 的 _changes 通过角度变化波峰检测动作转换点，但强制输出 恰好 8 段。当峰值不足时退回到均匀切分，当峰值过多时只取最强的 7 个。

核心矛盾在于：动作变化点 ≠ 音乐节拍点。一段编舞可能在 4 拍内平稳过渡，在另外 4 拍内发生 3 次快速切换。阈值 mean + 0.5×std 的 0.5 系数也是未经校准的经验值。

建议方案
  • 将 split_8beats.py 的 BPM 信息传入 score_dance.py，让音乐结构指导分段
  • 如果音频不可用，采用自适应分段：用角度变化的局部方差拐点取代全局阈值，段数不硬约束为 8
  • 引入最小段长约束（如 ≥ 1 秒），避免 0.1 秒长的无效段
P2
低置信度关键点线性插值产生幻影姿态
影响：遮挡帧的姿态还原质量
现象

score_dance.py:106-119 的 _fix 方法对低置信度关键点采用时间线性插值，窗口为 interp_window=3（跨度可达 6 帧、0.2 秒）。对快速旋转中被遮挡的关节（如转身时手臂被躯干挡住），线性插值会穿越身体中心线，产生"幻影手臂"。

建议方案
  • 不只用布尔阈值，同时利用 MediaPipe 提供的 visibility 和 presence 两个置信度分数
  • 对不同部位用不同窗口：肢体末端（手、脚）窗口缩小（遮挡多但变化快），躯干窗口可稍大
  • 加入骨骼长度约束验证：插值后的大臂长度不应突变超过 20%（物理上骨长固定）
  • 进阶：对下半身关键点用卡尔曼滤波替代线性插值
P2
split_8beats.py 纯固定 BPM，无音频节拍分析
影响：慢动作分段与音乐不同步
现象

split_8beats.py 完全依赖硬编码 BPM=120，假设恒定速度。实际舞蹈音乐 BPM 范围 70–200，且副歌前后的 ritardando/accelerando（渐慢/渐快）会导致时间切点逐渐漂移，最终整段偏移一个节拍以上。

建议方案
  • 集成 librosa.beat.beat_track 从音频提取真实的 beat onset 时间戳
  • 如果音频不可用，退而求其次：从参考视频的运动学周期性反推节奏（周期性角度变化 ≈ 节拍频率）
  • 支持手工标注 beat 时间点作为最高精度方案（裁判/教师标注一次，后续复用）
